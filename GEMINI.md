# GEMINI.md

## Working Environment
- This project is developed on a Chromebook using Linux.
- Assume limited local system access and avoid workflows that rely on heavy native installations.
- Prefer solutions that work in a browser-first, web-native environment.
- Avoid build pipelines, tooling, or dependencies that require complex setup.

## Platform & Approach
- Target web-based applications that run in a standard browser.
- Favor plain HTML, CSS, and JavaScript.
- Avoid frameworks or advanced tooling unless explicitly requested.
- Prefer approaches that can be tested easily with minimal local setup.

## Developer Profile
- The developer is a novice programmer with no formal computer science background.
- Use clear, simple language and avoid unnecessary jargon.
- Optimize for readability and understandability over cleverness or “best practice” purity.
- Explain reasoning when making technical decisions.

## Guidance Style
- Prioritize clarity, simplicity, and incremental progress.
- Prefer small, understandable steps over large or abstract changes.
- When multiple options exist, recommend the simplest viable approach first.
- Assume the goal is learning through building, not engineering perfection.

## Default Assumption
When in doubt, choose the solution that is:
- easier to understand
- easier to test
- easier to undo
